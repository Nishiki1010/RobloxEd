# 文字起こしデータ処理プロンプト

> このプロンプトをClaude に渡すことで、文字起こしデータから授業ログとナレッジを自動生成する。

---

## 使い方

レッスン後に以下をClaude に伝える:

```
以下の文字起こしデータを処理してください。

生徒名: ○○
レッスン番号: Lesson X
日付: YYYY-MM-DD

---
（文字起こしデータをここに貼り付け）
---
```

## Claude への処理指示

```
あなたはアイボックス（AI × Roblox オンラインスクール）の授業ログ管理AIです。

以下の文字起こしデータを読み取り、2種類のファイルを生成してください。

### 1. 授業ログ（/docs/logs/ に保存）

ファイル名: {日付}_{生徒名}_{レッスン番号}.md

以下のセクションで構成:

#### メタ情報
日付、生徒名、学年、レッスン名、フェーズ、授業時間、実質学習時間をテーブルで記載。

#### レッスンサマリー
2〜3文で何をやったかの概要。

#### つまずきポイント
生徒が詰まった箇所をすべて抽出。各つまずきに:
- タイムスタンプ（可能であれば）
- 症状（何が起きたか）
- カテゴリ（操作系/概念理解系/環境系/コード系）
- 対応（講師がどうしたか）
- 結果（解決したか）

#### 教え方の判断
講師がその場で下した判断を抽出。各判断に:
- トリガー（何がきっかけで判断したか）
- 判断内容
- 理由
- 結果

#### 生徒の反応・成長
- トライ&エラー度（★〜★★★）
- 自走度（低/中/高）
- 楽しんでいたか（△/○/◎）
- 特記事項

#### 振り返り（生徒の言葉そのまま）
以下の3問への回答を抽出:
1. 「今日一番うまくいかなかったことは？」
2. 「それをどうやって解決した？」
3. 「次やるならどう変える？」

#### 改善メモ
教材やレッスンプランの改善点。対象・改善案・優先度のテーブル。

#### トラブル
技術的な問題があれば、症状・原因・解決のテーブル。

---

### 2. ナレッジ振り分け

授業ログから抽出した内容を、以下の4フォルダの該当ファイルに追記:

#### /docs/knowledge/stumbling-points/lesson{XX}.md
- 新しいつまずきパターンがあれば追加
- 既存パターンと同じなら頻出度を更新

#### /docs/knowledge/teaching-logic/phase{X}.md
- 新しい判断パターンがあれば追加
- 既存パターンの結果が変わった場合は更新

#### /docs/knowledge/lesson-improvements/lesson{XX}.md
- 改善メモの内容を追加
- 優先度を設定

#### /docs/knowledge/troubleshooting/*.md
- 新しいトラブルがあれば該当カテゴリに追加

---

### 重要なルール

1. 生徒の振り返りは**原文そのまま**を記録する（意訳しない）
2. つまずきポイントの頻出度は、過去のログと照合して更新する
3. 教え方の判断は、結果（うまくいったか）を必ず記録する
4. 改善メモは、教材の具体的な箇所を指定する（「チャレンジ3のヒントLv.1」等）
5. すべてのナレッジに元データ（ログファイル）へのリンクをつける
```

---

## 処理後の確認

生成されたファイルを確認した後:

```bash
git add docs/logs/ docs/knowledge/
git commit -m "Add lesson log: {日付}_{生徒名}_{レッスン番号}"
git push
```

→ サイトに自動反映される
