## チャレンジ

### チャレンジ 1: 溶岩っぽいブロックを作ろう（5分）

**ミッション**: 「"危険な感じ"がするブロックを作ってみて。色、大きさ、形、自由にデザインしよう！」

**想定つまづき（講師用）**:
- 何色がいいかわからない → 「溶岩ってどんなイメージ？」→ 赤、オレンジ
- Material を知らない → まだ教えなくてOK。色だけで十分

**ヒント**:
- Lv.1: 「薄くて広いブロックにすると"床"っぽくなるよ」
- Lv.2: 「赤やオレンジにすると溶岩っぽいかも」
- Lv.3: 「Properties の Material を "Neon" にすると光って見えるよ！」（発展）

---

### チャレンジ 2: 踏んだらどうなる？コードを考えよう（15分）

**ミッション**: 「前回は"触ったら色が変わる"コードを書いたよね。今回は"触ったらキャラがやられる"コードを書きたい。どう変えればいいと思う？」

**まず考えさせる（講師用）**:
前回のコードを思い出させる:
```lua
part.Touched:Connect(function(hit)
    part.BrickColor = BrickColor.new("Really red")
end)
```
「このコードの "色を変える" 部分を "キャラのHPを0にする" に変えたいんだけど、どうすればいいかな？」

**書くコード（段階的に出す）**:

```lua
local part = script.Parent

part.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = 0
    end
end)
```

**想定つまづき（講師用）**:
- `hit` って何？ → 「触ったもの。プレイヤーの足とか手とかがここに入るよ」
- `hit.Parent` って何？ → 「足の"親"は誰？→ キャラクター全体だよね」
- `FindFirstChild("Humanoid")` → 「"Humanoid っていう部品を探して"って意味。キャラクターにはHumanoidが入ってるんだ」
- `if humanoid then` → 「"もし見つかったら" って意味。見つからなかったらスキップする」

**ヒント**:
- Lv.1: 「前回のコードがベースだよ。`Touched` の部分は同じ」
- Lv.2: 「`hit` は触ったもの。`hit.Parent` はそれが所属してるキャラクター」
- Lv.3: コードを1行ずつ一緒に書く

**超重要: `if` 文の初体験**:
「`if humanoid then` = "もし humanoid があったら"。これがプログラミングで一番大事な考え方の1つ。"もし〇〇なら△△する"」

> 公式: [Humanoid](https://create.roblox.com/docs/reference/engine/classes/Humanoid)

**ミッション**: 「Play を押して、溶岩ブロックの上を歩いてみて。何が起きる？」

**成功した瞬間の声かけ**:

### チャレンジ 4: 復活地点を設置しよう（10分）

**ミッション**: 「やられた後、どこに復活した？もっとちゃんとした復活地点を作りたくない？」

**想定つまづき（講師用）**:
- SpawnLocation の入れ方がわからない → 段階的にヒントを出す
- 復活地点が変な場所にある → 「Playして確認してみて。思った場所に復活する？」

**ヒント**:
- Lv.1: 「Roblox には"ここに復活する"っていう特別なパーツがあるよ。探してみて」
- Lv.2: 「Home タブの中を見てみて。Part 以外にも何かないかな？」
- Lv.3: 「Home → Spawn を押してみて。チェック柄のパーツが出てくるよ」

**SpawnLocation について**:
- 配置するだけで、プレイヤーの復活地点になる
- 好きな場所に移動できる
- 複数置くこともできる（ランダムで復活）

> 公式: [SpawnLocation](https://create.roblox.com/docs/reference/engine/classes/SpawnLocation)

### チャレンジ 5: ミニステージを作ろう！（30分）

**ミッション**: 「Lesson 2 の障害物コースに、今日作った"溶岩の罠"と"復活地点"を組み合わせて、もっと面白いステージにしよう！」

**やること**:

2. 足場の間に溶岩ブロック（罠）を置く

**講師の役割**:

**よくある展開と対応**:
| 生徒の状態 | 講師の対応 |

## 講師メモ（授業前3分で読む）

### この回の核心
Lesson 4 の本当のゴールは「溶岩を作ること」ではなく、**「if文（条件分岐）」の感覚を掴むこと**と、**前回学んだTouchedイベントを応用する体験**。「前にやったことをベースに新しいことを作る」というプログラミングの本質を体験させる。

### 超重要: hitとhumanoidの関係
`hit` → `hit.Parent` → `FindFirstChild("Humanoid")` の流れは最初は理解しにくい。Explorer で実際にキャラクターの構造を見せると分かりやすい。「キャラクターの中にHumanoidっていう部品があるんだよ。それがHP（体力）を管理してるんだ」

### Explorerでキャラクター構造を見せる
Playした状態で Explorer → Workspace → プレイヤー名 を開くと:
```
Player名
├── Head
├── Torso
├── Left Arm
├── Right Arm
├── Left Leg
├── Right Leg
├── Humanoid  ← これ！
└── ...
```
「足（Left Leg）がブロックに触る → 足の親はキャラクター → キャラクターの中にHumanoidがある → HealthをOにする」という流れを図で見せる。

### NG対応
- ❌ コード全体を最初に見せて「これを打って」
- ❌ `if` 文の説明を長々とする（体験が先）
- ❌ エラーを講師が直してしまう

### OK対応
- ✅ 「前回のコードをベースに変えてみよう」
- ✅ 「`if` は"もし〇〇なら"って意味。日常でも使うよね？」
- ✅ 「エラーの赤い文字、何て書いてある？一緒に読もう」
- ✅ 「Explorer でキャラクターの中身を見てみよう」
