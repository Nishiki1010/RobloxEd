### チャレンジ 2: 溶岩コードを自分で考えてみよう（15分）

**ミッション**: 「コインのコードを参考に、"触ったらやられる"コードを自分で書いてみよう！」

**講師の問いかけ**:
- 「コインのコードを思い出して。Touched で何かが起きるんだよね」
- 「コインは `Destroy()` で消した。今回は何をする？」
- 「キャラクターの"体力"を 0 にしたいんだけど...」

**生徒に考えさせる時間を取る（3〜5分）**

**書くコード**:

```lua
local lava = script.Parent

lava.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = 0
    end
end)
```

**コインとの比較（講師がガイド）**:

| | コインのコード | 溶岩のコード |
|---|---|---|
| Touched | 同じ | 同じ |
| if humanoid then | 同じ | 同じ |
| やること | `coin:Destroy()` | `humanoid.Health = 0` |

「ほら、ほとんど同じでしょ？違うのは最後の1行だけ。これがプログラミングの面白いところ。パターンを覚えれば、いろんなものが作れるんだ」

**講師の声かけ**:

### チャレンジ 5: 発展 — ダメージ罠を作ろう（15分）

**ミッション**: 「即死じゃなくて、少しずつダメージを受ける罠も作ってみよう」

```lua
local damagePart = script.Parent

damagePart.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = humanoid.Health - 25
    end
end)
```

**説明**: 「`Health = 0` は即死。`Health = Health - 25` は体力を25減らす。デフォルトの体力は100だから、4回触ったらやられるね」

**自由に試す**:
- ダメージ量を変えてみる（10? 50? 99?）
- 即死罠とダメージ罠を混ぜて配置
- 「どっちが面白い？テストして決めよう」

## 講師メモ（授業前3分で読む）

### この回の核心
**「コインのコードパターンを溶岩に応用する」**体験。同じ Touched + Humanoid のパターンで、やることだけ変えれば別の機能が作れる。パターン認識力がプログラミングの核。

### 超重要: 自分で書かせる
チャレンジ 2 で「コインのコードを参考に自分で書いてみて」と促す。最初から答えを見せない。5分考えさせて、詰まったらヒントを出す。

### ゲームバランスの概念
「難しすぎず、簡単すぎず」を生徒自身に判断させる。これもトライ&エラーの一種。

### NG対応
- ❌ 溶岩コードを最初から見せる
- ❌ 罠の配置場所を講師が決める

### OK対応
- ✅ 「コインのコードとどこが似てる？どこが違う？」
- ✅ 「自分でクリアできる？テストしてみて」
- ✅ 「パターンを覚えれば何でも作れる！」
