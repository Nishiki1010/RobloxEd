### チャレンジ 3: BGM を追加しよう（10分）

**ミッション**: 「ゲームに音楽を追加して、雰囲気を出そう！」

**手順**:
1. Workspace の中に Sound を追加
2. SoundId を設定（Roblox の Asset ID を使う）
3. Looped にチェック（ループ再生）
4. Playing にチェック（自動再生）

**コードでやる場合**:
```lua
local sound = Instance.new("Sound")
sound.SoundId = "rbxassetid://1234567890"  -- 好きな音楽のID
sound.Looped = true
sound.Volume = 0.5
sound.Parent = workspace
sound:Play()
```

**音楽の探し方**: 「Roblox の Toolbox → Audio で検索できるよ」

**講師判断**: 著作権のある音楽は使わないように注意。Roblox 公式の音源を使う。

### チャレンジ 4: 効果音を追加しよう（10分）

**ミッション**: 「コインを取ったとき、チャリン！って音が鳴るようにしよう」

**コインのコードに追加**:

```lua
local coin = script.Parent
local scoreValue = workspace:FindFirstChild("Score")
local totalCoins = 5

-- 効果音を準備
local collectSound = Instance.new("Sound")
collectSound.SoundId = "rbxassetid://6895079853"  -- コイン音
collectSound.Parent = coin

coin.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        scoreValue.Value = scoreValue.Value + 1
        collectSound:Play()
        task.wait(0.2)  -- 音が鳴り終わるのを少し待つ
        coin:Destroy()
    end
end)
```

**ポイント**: `task.wait(0.2)` で音が鳴ってから消す。じゃないと消えた瞬間に音も消える。

## 講師メモ（授業前3分で読む）

### この回の核心
**「自分が作ったゲームが完成した！」という達成感**が最大の学び。技術的に新しいことは少ない（音を入れる程度）。主役はデバッグと仕上げ。

### 超重要: 通しプレイで「完成」を実感させる
最初にチェックリストで通しプレイ → 問題を発見 → 修正。この「最後の仕上げ」がゲーム開発の醍醐味。そしてそれは今まで学んだことの集大成。

### 講師がプレイする
最後に講師がゲームをプレイする。「ここ難しい！」「おお、ここにコイン隠してたの！」と大きくリアクションする。作った人に遊んでもらう体験は何よりのモチベーション。
