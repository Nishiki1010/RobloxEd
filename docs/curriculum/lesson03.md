「前回は障害物コースを作ったよね。ブロックを並べて、Play して、調整して。今日からは"コード"を使うよ。コードを書くとブロックに"命令"を出せるんだ」

## チャレンジ

### チャレンジ 1: プログラミングって何？（10分）

**ミッション**: 「コンピュータと話す方法を見つけよう」

**講師が説明（対話形式で）**:

「普段、友達に何かお願いするとき、どうする？」→「言葉で伝えるよね」

「コンピュータも同じ。でもコンピュータは日本語がわからないから、コンピュータがわかる言葉で伝えるんだ。それが**プログラミング**」

「Roblox では **Luau**（ルアウ）っていう言葉を使うよ。今日はその最初の一歩」

**たとえ話**:
- プログラミング = コンピュータへの「お手紙」
- スクリプト = お手紙を入れる「封筒」
- 実行（Play） = お手紙を「届ける」

---

### チャレンジ 2: スクリプトを入れてみよう（5分）

**ミッション**: 「ブロックの中に"お手紙"を入れてみよう」

**想定つまづき（講師用）**:
- スクリプトの入れ方がわからない → 段階的ヒント
- Explorer が見つからない → View → Explorer

**ヒント**:
- Lv.1: 「まずブロックを1個作って。それに命令書を入れるよ」
- Lv.2: 「ブロックの名前が右の Explorer に出てるよね。その横の ＋ を押してみて」
- Lv.3: 「＋ を押したら "Script" を探して選んでみて」

**確認ポイント**: Explorer で Part の下に Script が入ったことを見せる。「ほら、ブロックの中にお手紙が入ったよ」

### チャレンジ 3: Hello World!（10分）

**ミッション**: 「コンピュータに最初のメッセージを送ってみよう！」

**やること**:
1. スクリプトをダブルクリックして開く
2. 最初から書いてある文字を全部消す
3. 以下を書く:

```lua
print("Hello World!")
```

4. Output ウィンドウを開く（View → Output）
5. Play を押す

**想定つまづき（講師用）**:
- Output が表示されない → View → Output にチェック
- 何も表示されない → 「スクリプトはブロックの中に入ってる？ Explorer で確認」
- タイプミス → 「エラーの赤い文字を見てみよう」

**ヒント**:
- Lv.1: 「`print` って打ってみて。printは"画面に表示して"っていう命令だよ」
- Lv.2: 「カッコの中に、ダブルクォーテーション（"）で囲んだ文字を入れるよ」
- Lv.3: 一緒に1文字ずつ打つ

**成功した瞬間**: 「見て！Output に "Hello World!" って出たでしょ？これ、自分が書いたコードが動いたんだよ！」

**超重要**: プログラマーが一番最初に書くのは世界中どこでも `Hello World!`。「君も今、世界中のプログラマーと同じことをやったんだよ」

### チャレンジ 4: いろんなメッセージを送ってみよう（10分）

**ミッション**: 「print を使って、いろんなものを表示させてみよう！」

**やってみよう**:

```lua
-- 自分の名前を表示
print("ぼくの名前は太郎です")

-- 計算もできる！
print(1 + 1)
print(100 - 37)
print(3 * 4)

-- 文字と数字を組み合わせる
print("答えは " .. 2 + 3)
```

**講師の声かけ**:
- 「`--` で始まる行はコメントって言って、コンピュータは無視するよ。自分用のメモだね」
- 「計算もできるの！？って思った？ print の中に計算式を書くと、答えが出るんだよ」
- 「エラー出た？OK！何て書いてある？」

**自由に試させる**: 好きなメッセージ、好きな計算を print してみる

---

### チャレンジ 5: 変数 — 名前をつけて覚えさせよう（15分）

**ミッション**: 「コンピュータに"覚えて"って言ってみよう」

**導入**: 「毎回 `print("太郎")` って書くの面倒じゃない？コンピュータに"名前を覚えて"って言えるよ」

```lua
local name = "太郎"
print(name)
print("こんにちは、" .. name .. "さん！")
```

**たとえ話**: 「変数は"名前のついた箱"。箱に `name` って名前をつけて、中に "太郎" を入れる。使いたいときは箱の名前を言えば中身が出てくる」

**やってみよう**:

```lua
local age = 11
local school = "アイボックス"
print(name .. "は" .. age .. "歳で、" .. school .. "に通っています")
```

**チャレンジ**: 「自分のプロフィールを変数で作って表示してみよう！」

```lua
local name = "（自分の名前）"
local age = （自分の年齢）
local hobby = "（好きなこと）"
print(name .. "、" .. age .. "歳、趣味は" .. hobby)
```

**想定つまづき（講師用）**:
- `local` を忘れる → 「最初に `local` って書くのがルールだよ」
- 文字列を `"` で囲み忘れる → 「文字は必ずダブルクォーテーションで囲むよ」
- `..` の使い方 → 「点を2つ（..）で文字をくっつけられるよ」

---

### チャレンジ 6: ブロックに触ったらメッセージ！（15分）

**ミッション**: 「ブロックに触ったら Output に"タッチ！"って出るようにしてみよう」

**導入**: 「さっきまでは Play したら勝手に表示されたよね。今度は"触ったときだけ"表示させたい。そういう仕組みがあるよ」

**書くコード**:

```lua
local part = script.Parent

part.Touched:Connect(function()
    print("タッチ！")
end)
```

**1行ずつ説明（講師がガイド）**:
- `local part = script.Parent` → 「このスクリプトが入ってるブロックを `part` って名前にする」
- `part.Touched:Connect(function()` → 「`part` に何かが触ったら...」
- `print("タッチ！")` → 「"タッチ！"って表示する」
- `end)` → 「おわり」

**やってみよう（自由に試す）**:
- メッセージを変えてみる: `print("いたっ！")`, `print("見つけた！")`
- 変数を使ってみる:
```lua
local message = "ブロックに触ったよ！"
part.Touched:Connect(function()
    print(message)
end)
```

**ポイント**: ここでは **print だけ**。色を変えたり消したりはしない。「触ったら何かが起きる」という仕組みを理解させることが目的。

> 公式: [Touched Event](https://create.roblox.com/docs/reference/engine/classes/BasePart#Touched)

## 講師メモ（授業前3分で読む）

### この回の核心
Lesson 3 のゴールは「print と変数と Touched を覚えること」ではなく、**「自分が書いた文字がコンピュータに伝わった！」という感動体験**。プログラミングの最初の一歩を「楽しい！もっとやりたい！」で終わらせる。

### 超重要: Output ウィンドウを先に開かせる
print を書いても Output が見えなかったら何も起きていないように見える。**チャレンジ 3 の前に必ず View → Output を開かせる**。

### エラーとの付き合い方（ここで文化を作る）
- 初めてのコードなので、タイプミスは必ず起きる
- エラーが出たら「やった！エラーだ！コンピュータが"ここ違うよ"って教えてくれてるよ」
- 絶対に「間違ってるよ」と言わない。「コンピュータからのお手紙を読んでみよう」

### 次回への接続
最後の振り返りで「次回から"コイン集めアドベンチャー"を作る」と予告する。「今日覚えた print と Touched は、次回以降のゲーム作りでめちゃくちゃ使うよ！」

### NG対応
- ❌ コードを全部見せて「これを打って」
- ❌ 変数の厳密な説明をする（メモリ、型など）
- ❌ エラーを講師が直してしまう

### OK対応
- ✅ 「print は"表示して！"って命令だよ」
