# Lesson 3: 魔法のコードを書こう

**Phase 1 | コード: あり（初スクリプト） | 80分 | 初心者対応**

---

## 今日のゴール

> 「触ると色が変わるブロック」を作る！
> はじめてのプログラミング（スクリプト）に挑戦しよう。

**完成イメージ**: プレイヤーが触ると色が変わるブロック。魔法みたいでしょ？

---

## 前回の復習（3分）

「前回は障害物コースを作ったよね。今日はもっとすごいことをするよ。ブロックに"魔法"をかけるんだ。触ったら色が変わるブロックを作るよ！」

## チャレンジ

### チャレンジ 1: 魔法をかけるブロックを用意しよう（3分）

**ミッション**: 「まず、魔法をかけたいブロックを1個作って、好きな色にしてみて」

**ヒント**:

- Lv.2: 「Home → Part → Block。色も変えてね」
- Lv.3: 「Anchored もチェックしておこう」

### チャレンジ 2: スクリプトって何？入れてみよう（10分）

**ミッション**: 「ブロックに"命令書"を入れることができるよ。どうやって入れるか、探ってみて！」

**想定つまづき（講師用）**:
- スクリプトの入れ方がわからない → 「ブロックを右クリックしてみて。何かメニューが出ない？」
- Explorer が見つからない → 「右側のパネルを見てみて」

**ヒント**:
- Lv.1: 「ブロックを選んだ状態で、右クリックしてみて」
- Lv.2: 「メニューの中に "Insert Object" か "Script" ってないかな？」
- Lv.3: 「ブロックの ＋ ボタンを押して、"Script" を選んでみて」

**講師判断**: Explorer で Part の下に Script が入ったのを見せる。「ブロックの中に命令書が入ったよ。この命令書に書いたことを、ブロックが実行するんだ」と説明。

> 公式: [Creating Scripts](https://create.roblox.com/docs/scripting/scripts)

---

### チャレンジ 3: はじめてのコードを書こう（15分）

**ミッション**: 「スクリプトを開くと文字が書いてあるよね。それを全部消して、代わりにこの"魔法の呪文"を書いてみて」

**書くコード（講師が画面共有またはチャットで共有）**:

```lua
local part = script.Parent

part.Touched:Connect(function(hit)
    part.BrickColor = BrickColor.new("Really red")
end)
```

**想定つまづき（講師用）**:
- タイプミス → 「エラーが出た？どこが違うか、一文字ずつ比べてみよう」← 最高のトライ&エラー体験
- 大文字小文字 → 「プログラミングは大文字と小文字が違うと動かないよ。BrickColor の B と C は大文字」
- print("Hello world!") が残っている → 「最初から入ってるやつは消して大丈夫だよ」

**ヒント**:
- Lv.1: 「1行ずつゆっくり書いてみよう。急がなくてOK」
- Lv.2: 「エラーが出たら Output ウィンドウを見てみて。何行目がおかしいか教えてくれるよ」
- Lv.3: 講師が画面共有してゆっくり見せる

**超重要: エラーは"いいこと"**:
「エラーが出た！やった！これはコンピュータが"ここが違うよ"って教えてくれてるんだ。エラーが出ない人はいない。プロでもエラーは出る。大事なのは"エラーを読んで直す"こと」

---

### チャレンジ 4: 魔法を試そう！（5分）

**ミッション**: 「Play を押して、ブロックに触ってみて！何が起きる？」

**想定つまづき（講師用）**:
- 何も起きない → 「スクリプトはブロックの中に入ってる？Explorer で確認してみて」
- エラーが出る → 「Output ウィンドウを一緒に見よう。何て書いてある？」

**ヒント**:
- Lv.1: 「キャラクターをブロックに向かって歩かせてみて」
- Lv.2: 「触った？色が変わった？」
- Lv.3: エラーの場合: 「Output を開いて（View → Output）、赤い文字を読んでみよう」

**成功した瞬間の声かけ**:
「すごい！自分で書いたコードが動いた！これがプログラミングだよ。今、自分で"魔法"を作ったんだ」

---

### チャレンジ 5: コードを読み解こう（10分）

**ミッション**: 「動いたのはすごいけど、なんで動いたんだろう？1行ずつ、何をしてるか考えてみて」

**一緒に読み解く（講師がガイド）**:

```lua
local part = script.Parent
-- 「このスクリプトが入っているブロック」を part という名前にする

part.Touched:Connect(function(hit)
-- 「part に何かが触ったら...」
    part.BrickColor = BrickColor.new("Really red")
    -- 「ブロックの色を赤に変える」
end)
-- 「おわり」
```

**問いかけ**:
- 「`script.Parent` って何だと思う？」→ スクリプトの親 = ブロック
- 「`Touched` って何だと思う？」→ 触った
- 「`Really red` を変えたらどうなると思う？」→ 次のチャレンジへ

> 公式: [Touched Event](https://create.roblox.com/docs/reference/engine/classes/BasePart#Touched)

### チャレンジ 6: 魔法をカスタマイズしよう！（25分）

**ミッション**: 「色を赤以外に変えてみて！他にどんな色が使える？」

**やってみよう（自由に試す時間）**:

1. **色を変える**: `"Really red"` を `"Bright blue"`, `"Lime green"`, `"Hot pink"` などに変更
2. **透明にする**: `part.Transparency = 0.5` を追加（半透明！）
3. **サイズを変える**: `part.Size = Vector3.new(10, 10, 10)` を追加（巨大化！）

**使える色の例**:
| コード名 | 色 |
|---------|-----|
| "Really red" | 赤 |
| "Bright blue" | 青 |
| "Lime green" | 黄緑 |
| "Hot pink" | ピンク |
| "New Yeller" | 黄色 |
| "Royal purple" | 紫 |

**講師の役割**:
- 「他にも変えられるものがあるよ」とだけ言って、自分で試させる
- 「Transparency って何だと思う？」→ 数字を変えて試させる
- 「エラーが出た？何て書いてある？」→ 自分で直させる

**発展（早い生徒向け）**:
- 触るたびにランダムな色に変わるようにしてみよう
- 複数のブロックに違う魔法をかけてみよう

## 講師メモ（授業前3分で読む）

### この回の核心
Lesson 3 の本当のゴールは「Touchedイベントを理解すること」ではなく、**「コードを書く → エラーが出る → 読んで直す → 動く！」の体験をさせること**。初めてのプログラミングだからこそ、エラーを"失敗"ではなく"ヒント"として受け取る習慣をここで作る。

### 超重要: エラーを歓迎する文化を作る
最初のエラーが出た瞬間が最大のチャンス。「やった！エラー出た！」くらいのテンションで対応する。ここでの講師の反応が、生徒のエラーに対する姿勢を決める。

### Output ウィンドウの出し方
View → Output で表示。エラーは赤い文字で出る。行番号も表示されるので「〇行目を見てみて」と指示できる。

### NG対応
- ❌ 「ここが間違ってるよ」とすぐ教える
- ❌ タイプミスを講師が直してしまう
- ❌ 「エラー出てるよ」と不安にさせるトーンで言う

### OK対応
- ✅ 「エラー出たね！何て書いてある？一緒に読んでみよう」
- ✅ 「1行ずつ比べてみよう。どこが違う？」
- ✅ 「動いた！！すごい！自分で書いたコードだよ！」
- ✅ 「エラーはコンピュータからのお手紙。"ここ直して"って教えてくれてるんだ」

### BrickColor の色名
正しい色名を使わないとエラーになる。公式リストを参照: [BrickColor List](https://create.roblox.com/docs/reference/engine/datatypes/BrickColor)
