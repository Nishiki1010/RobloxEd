## Phase の考え方

| Phase | レッスン | テーマ | 講師の役割 |
|-------|---------|--------|-----------|
| **Phase 1** | 1〜3 | 触って慣れる＋最初のコード | ヒントを多めに。「楽しい！」が最優先 |
| **Phase 2** | 4〜10 | 1つのゲームを作りながら学ぶ | 前回の知識をベースに新しいことを追加。「なぜこのコードが必要か」が常に明確 |
| **Phase 3** | 11〜14 | 応用と自分の企画 | ヒントを減らす。「自分で調べる」を促す |
| **Phase 4** | 15〜18 | オリジナルゲーム開発 | 見守る。生徒が主導 |

> Phase 2 のポイント: 毎回「自分のゲームに1つ機能が追加される」ので、「なぜこのコードを学ぶのか」が常に明確。
