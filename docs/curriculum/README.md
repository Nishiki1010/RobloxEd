## Phase の考え方

| Phase | レッスン | テーマ | 講師の役割 |
|-------|---------|--------|-----------|
| **Phase 1** | 1〜6 | 触って慣れる | ヒントを多めに出す。まず「楽しい」を体験させる |
| **Phase 2** | 7〜14 | コードで仕組みを作る | ヒントを減らす。「自分で調べる」を促す |
| **Phase 3** | 15〜18 | 自分のゲームを作る | 見守る。生徒が主導。講師は質問に答えるだけ |

> Phase が進むにつれて、講師の介入を減らしていく。これがトライ&エラーの自走力を育てる設計。
