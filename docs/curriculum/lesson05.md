## チャレンジ

### チャレンジ 1: コインを作ろう（10分）

**ミッション**: 「"コイン"っぽいパーツを作ってみて！」

**ヒント**:
- Lv.1: 「コインってどんな形？丸くて薄いよね」
- Lv.2: 「Part → Cylinder（円柱）を使って、薄くすればコインっぽくなるよ」
- Lv.3: 「色を黄色に、Material を Neon にすると光ってキレイだよ」

**やること**:
1. Part → Cylinder で円柱を作る
2. Scale で薄くする（コインの形に）
3. 色を黄色（`New Yeller` や `Bright yellow`）に
4. Material を `Neon` にすると光る
5. Anchored にチェック
6. Explorer で名前を `Coin` に変更

**名前を変える理由**: 「ブロックに名前をつけておくと、あとでコードを書くとき分かりやすいよ」

---

### チャレンジ 2: Touched の復習 — まず print から（5分）

**ミッション**: 「Lesson 3 を思い出して、コインに触ったら"ゲット！"って表示させよう」

```lua
local coin = script.Parent

coin.Touched:Connect(function()
    print("コインゲット！")
end)
```

**確認**: Play → コインに触る → Output に "コインゲット！" が出る

「OK！触ったら反応するようになったね。じゃあ今度は print じゃなくて"消す"をやろう」

---

### チャレンジ 3: 触ったらコインが消える！（15分）

**ミッション**: 「print の代わりに、コインを消すコードを書いてみよう」

**導入**: 「"コインゲット！"って表示されるのはいいけど、コインが残ってたらおかしいよね。取ったら消えてほしい」

**書くコード**:

```lua
local coin = script.Parent

coin.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        print("コインゲット！")
        coin:Destroy()
    end
end)
```

**新しい部分を1つずつ説明**:

| コード | 意味 |
|--------|------|
| `function(hit)` | `hit` = 触ったもの（プレイヤーの足とか手） |
| `hit.Parent` | 足の"お父さん" = キャラクター全体 |
| `FindFirstChild("Humanoid")` | 「Humanoid っていう部品ある？」と聞く |
| `if humanoid then` | 「もし見つかったら（= プレイヤーが触ったなら）」 |
| `coin:Destroy()` | 「コインを消す！」 |

**たとえ話**: 「なんで `if humanoid then` が必要かというと、ブロック同士がぶつかっただけでも Touched が反応しちゃうから。"人間が触ったときだけ"消えるようにしたいんだ」

**想定つまづき（講師用）**:
- `hit` と `hit.Parent` の関係がわからない → Explorer で Play 中のキャラクター構造を見せる
- `:Destroy()` のコロンを忘れる → 「ドットじゃなくてコロン（:）だよ」
- 触ってないのにエラー → スペルミスの可能性大。一緒に Output を見る

### チャレンジ 4: コインをたくさん配置しよう（15分）

**ミッション**: 「コインを5個以上、フィールドのいろんな場所に配置しよう！」

**やること**:
1. 動作確認済みのコインを選択
2. Ctrl+D（Mac: Cmd+D）でコピー
3. フィールドの面白い場所に配置
   - 高い場所（登らないと取れない）
   - 隠れた場所（トンネルの中）
   - 道の途中（進むと自然に取れる）
4. 全部テストプレイで取れるか確認

**講師の声かけ**:
- 「簡単に取れるのと、ちょっと工夫がいるのを混ぜると面白いよ」
- 「何個置いた？全部取れた？」
- 「Play して全部集めてみよう！」

**ポイント**: スクリプトごとコピーされるので、全部のコインが触ると消える。「1個書いたコードが全部に効いてる！これがプログラミングの力だよ」

### チャレンジ 5: 自由カスタマイズ（15分）

**ミッション**: 「コインをもっとカッコよくしよう。メッセージも変えてみよう！」

**アイデア**:
- コインの色をバラバラにする（金・銀・銅？）
- メッセージを変える（`print("レアコイン発見！")`）
- 音を想像する（「取ったとき"チャリン"って鳴ったらいいよね。それは後の回でやるよ！」）

**発展（早い生徒向け）**:
- 取ったら print で何個目かわかるようにする: `print("1個目ゲット！")`
- 「でもこれ、全部手動で書かないといけないよね？もっとスマートな方法がある。次回やるよ！」→ Lesson 6（変数でカウント）への布石

## 講師メモ（授業前3分で読む）

### この回の核心
Lesson 3 の print / Touched を「ゲームの機能」として実用化する初回。**「自分が書いたコードで、自分のゲームにコイン収集機能が入った！」**という体験がゴール。

### 超重要: Lesson 3 との接続
チャレンジ 2 で print からスタートし、チャレンジ 3 で Destroy に発展させる。「前回やったことをベースに新しいことを作る」この感覚がプログラミング学習の核。

### hit と Humanoid の説明
Explorer で Play 中にキャラクター構造を見せるのが最も効果的。「足が触った → 足の親はキャラクター → キャラクターの中に Humanoid がある」を図で見せる。

### NG対応
- ❌ コード全体を最初に見せる（print → Destroy の段階を飛ばさない）
- ❌ 「Humanoidは人型キャラクターの...」と用語を説明しすぎる

### OK対応
- ✅ 「前回の print を覚えてる？あれをベースに変えるだけだよ」
- ✅ 「Humanoid = "人間かどうかチェック"。それだけ覚えればOK」
- ✅ 「1個書いたら全部に効く。これがプログラミングの力！」
