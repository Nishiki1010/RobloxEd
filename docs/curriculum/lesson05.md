## 生徒用チャレンジ

### チャレンジ 1: コインを作ろう（10分）

**ミッション**: 「"コイン"っぽいパーツを作ってみて！」

**ヒント**:
- Lv.1: 「コインってどんな形？丸くて薄いよね」
- Lv.2: 「Part → Cylinder で円柱を出して、Scale で薄くしよう」
- Lv.3: 「色を黄色（Bright yellow）に、Material を Neon にすると光るよ」

**やること**:
1. Part → Cylinder で円柱を作る
2. Scale で薄くする
3. 色を黄色に変える
4. Material を Neon に（光る！）
5. Anchored にチェック
6. Explorer で名前を `Coin` に変更

**名前を変える理由**: 「名前をつけておくと、コードを書くとき分かりやすいよ」

### チャレンジ 2: Touched の復習 — まず print（5分）

**ミッション**: 「Lesson 3 を思い出して、コインに触ったら"ゲット！"って表示させよう」

```lua
local coin = script.Parent

coin.Touched:Connect(function()
    print("コインゲット！")
end)
```

**テスト**: Play → コインに触る → Output に "コインゲット！"

「OK！反応するようになったね。じゃあ今度は print じゃなくて"消す"をやろう」

---

### チャレンジ 3: 触ったらコインが消える！（20分）

**ミッション**: 「コインを取ったら消えるようにしよう」

**導入**: 「取ったのにコインが残ってたらおかしいよね。消えてほしい」

**書くコード**:

```lua
local coin = script.Parent

coin.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        print("コインゲット！")
        coin:Destroy()
    end
end)
```

**新しい部分を説明**:

| コード | 意味 | たとえ |
|--------|------|--------|
| `function(hit)` | 触ったもの | 「触ったのは誰？」を `hit` に入れる |
| `hit.Parent` | 触ったものの"親" | 足の親 = キャラクター全体 |
| `FindFirstChild("Humanoid")` | Humanoid を探す | 「人間ですか？」とチェック |
| `if humanoid then` | 見つかったら | 「人間が触った場合だけ」実行 |
| `coin:Destroy()` | コインを消す | 消滅！ |

**なぜ `if humanoid then` が必要？**:

「ブロック同士がぶつかっても Touched は反応しちゃう。"人間が触ったときだけ"消えるようにしたいんだ」

**想定つまづき**:
| エラー | 原因 | 対応 |
|--------|------|------|
| hit.Parent が nil | プレイヤー以外が触った | if humanoid then で防いでいる |
| `:Destroy()` でエラー | `.Destroy()` と書いている | 「ドットじゃなくてコロン(:)だよ」 |
| 触っても消えない | スクリプトが Part の外にある | Explorer で確認 |

**Explorer で構造を見せる（Play中）**:
```
Workspace
└── Coin
    └── Script  ← ここにスクリプトがある

Players
└── プレイヤー名
    ├── Head
    ├── Torso
    ├── Left Leg  ← hit = これが触った
    ├── Right Leg
    ├── Humanoid  ← これを探してる！
    └── ...
```

「足が触った → 足の親はキャラクター → キャラクターの中に Humanoid がある → OK、人間だ！ → コイン消す」

### チャレンジ 4: コインをたくさん配置しよう（15分）

**ミッション**: 「コインを5個以上、フィールドのいろんな場所に配置しよう！」

**やること**:
1. 動作確認済みのコインを選択
2. Ctrl+D（Mac: Cmd+D）でコピー
3. フィールドに配置:
   - 道の途中（自然に取れる）
   - 高い場所（登らないと取れない）
   - 隠れた場所（トンネルの中）
4. 全部テストプレイで取れるか確認

**ポイント**: 「スクリプトごとコピーされるから、全部のコインが触ると消える。**1個書いたコードが全部に効く**。これがプログラミングの力！」

**講師の声かけ**:
- 「簡単に取れるのと、ちょっと工夫がいるのを混ぜると面白いよ」
- 「何個置いた？Play して全部集めてみよう！」
