# 授業ログ

## データフロー

```
文字起こしデータ（生テキスト）
    ↓ Claude が自動整形
授業ログ（この /logs フォルダ）
    ↓ 4カテゴリに抽出・振り分け
ナレッジベース（/knowledge フォルダ）
    ├── stumbling-points/    つまずきポイント
    ├── teaching-logic/      教え方の判断ロジック
    ├── lesson-improvements/ レッスンプラン改善
    └── troubleshooting/     トラブルシューティング
```

## ファイル命名規則

```
{日付}_{生徒名}_{レッスン番号}.md
例: 2026-03-09_tanaka_L01.md
```

## ログの構造

各ログファイルは以下のセクションで構成される:

1. **メタ情報** — 日付、生徒、レッスン番号、実施時間
2. **レッスンサマリー** — 何をやったかの概要
3. **つまずきポイント** — 生徒が詰まった箇所とその対応
4. **教え方の判断** — 講師がその場で下した判断
5. **生徒の反応・成長** — トライ&エラー度、自走度
6. **振り返り** — 生徒自身の言葉
7. **改善メモ** — 教材やレッスンプランの改善点
8. **トラブル** — 技術的な問題とその解決法

## 使い方

1. レッスン後、文字起こしデータをClaude に渡す
2. Claude が上記フォーマットでログを生成
3. `/logs` に保存
4. 同時に該当するナレッジを `/knowledge` の各フォルダに振り分け
5. git push → サイトに反映
