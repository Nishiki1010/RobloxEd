# ナレッジベース

授業ログから自動抽出された知見を、4つのカテゴリに蓄積・管理する。

---

## カテゴリ

### [つまずきポイント](knowledge/stumbling-points/)

生徒が実際に詰まった箇所の記録。同じ箇所で複数の生徒が詰まった場合、教材自体の改善が必要なサイン。

- レッスン番号・フェーズ別に整理
- 症状 → 原因 → 対処法 のフォーマット
- 頻出度（何人の生徒で発生したか）を追跡

### [教え方の判断ロジック](knowledge/teaching-logic/)

講師がレッスン中に下した判断の記録。「こういう時はこうする」のパターン集。

- トリガー条件（生徒がこうしたら）
- 判断内容（講師はこう対応する）
- 結果（うまくいったか）
- 次回への教訓

### [レッスンプラン改善](knowledge/lesson-improvements/)

教材やレッスンの流れに対する改善提案。実際の授業から見えた改善点。

- 対象レッスン
- 現状の問題
- 改善案
- 優先度

### [トラブルシューティング](knowledge/troubleshooting/)

技術的な問題とその解決法。Roblox Studio のバグ、ネットワーク問題、環境設定など。

- 症状
- 原因
- 解決手順
- 予防策

---

## 蓄積の仕方

1. 授業ログから Claude が自動抽出
2. 既存のナレッジと重複する場合はマージ（頻出度を更新）
3. 新しいパターンの場合は新規ファイルとして追加
4. 定期的にレビューして教材（`/curriculum`）に反映
