# Phase 1: 教え方の判断ロジック

> 授業ログから抽出した、Phase 1（Lesson 1-6）での判断パターン。

---

### 生徒が選択肢を広げすぎようとした時 → 範囲を制限する

- **トリガー**: 生徒が Part ドロップダウンで Block 以外（Sphere 等）を選ぼうとした
- **判断**: 「いい質問！でも今日は Block だけ使うよ。丸いのは次回以降のお楽しみ」
- **理由**: 初回は選択肢を絞ることで、基本操作に集中させる。興味は殺さず次回への期待に変える
- **結果**: ✅ 生徒は納得して進めた
- **適用レッスン**: Lesson 1〜2
- **元データ**: [2026-03-09_sample_L01](logs/2026-03-09_sample_L01.md)

---

### 生徒が "正しくない" ことをして楽しんでいる時 → 止めない

- **トリガー**: 生徒が Scale で巨大なブロックを作って笑っていた
- **判断**: 止めずに楽しませた。問題が起きてから（Baseplate はみ出し等）教える
- **理由**: 創造の楽しさが最優先。問題が起きてから教える方が記憶に残る
- **結果**: ✅ 後で自分から「大きすぎた」と気づいて直した
- **適用レッスン**: Phase 1 全般
- **元データ**: [2026-03-09_sample_L01](logs/2026-03-09_sample_L01.md)

---

### 生徒が指示待ちモードの時 → 自由制作に切り替える

- **トリガー**: 手順指示のチャレンジでは受動的だが、自由制作で急に活発になった
- **判断**: この生徒には早めに自由制作に入らせる
- **理由**: 手順指示より自由度がある方が合う生徒がいる
- **結果**: ✅ 自由制作で「すごい！」と声を上げて楽しんでいた
- **適用レッスン**: Phase 1 全般（生徒の性格による）
- **元データ**: [2026-03-09_sample_L01](logs/2026-03-09_sample_L01.md)
