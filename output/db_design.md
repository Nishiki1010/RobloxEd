# アイボックス DB 設計書

---

## DB1: 授業ログDB

レッスン後に1レコード追加するだけ。5つの旧DBの役割を1つに集約。

### カラム定義

| # | カラム名 | 型 | 選択肢/設定 | 説明 |
|---|---------|------|-----------|------|
| 1 | タイトル | Title | 自動: 「{日付}_{生徒名}_{レッスン番号}」 | レコード名 |
| 2 | 日付 | Date | - | 実施日 |
| 3 | 生徒 | Relation | → 生徒カルテDB | 誰のレッスンか |
| 4 | レッスン番号 | Select | Lesson 1〜Lesson 18 | どのレッスンか |
| 5 | フェーズ | Formula | Lesson 1-6→Phase 1, 7-14→Phase 2, 15-18→Phase 3 | 自動算出 |
| 6 | つまづいた箇所 | Text | - | 何で詰まったか（自由記述）|
| 7 | 生徒の反応 | Select | 自力で解決 / ヒントで解決 / 講師が介入 / 未解決 | トライ&エラーの度合い |
| 8 | トライ&エラー度 | Select | ★ / ★★ / ★★★ | 自分から試す姿勢の評価 |
| 9 | 振り返りの回答 | Text | - | 生徒自身の振り返り（3つの質問への回答）|
| 10 | 気づいたこと | Text | - | 講師メモ（次回への引き継ぎ）|
| 11 | 教材改善メモ | Text | - | 「この説明を変えた方がいい」等 |
| 12 | 実質学習時間（分） | Number | - | 実際に手を動かしていた時間 |

### 旧DBとの対応表

| 旧DB | 新DBでのカバー方法 |
|------|------------------|
| レッスンフィードバックDB | → 授業ログDB のレコード自体 |
| つまづきポイントDB | → 「つまづいた箇所」カラム |
| トラブルシューティングガイドDB | → 「つまづいた箇所」+レッスンページの「想定つまづき」に集約 |
| 教え方の判断ロジックDB | → レッスンページの「講師判断」セクションに集約 |
| レッスンプラン改善ログDB | → 「教材改善メモ」カラム |

### ビュー設計

| ビュー名 | 種類 | フィルタ/ソート | 用途 |
|---------|------|--------------|------|
| 全ログ | Table | 日付降順 | デフォルト一覧 |
| 生徒別 | Table | 生徒でグループ化 | 特定生徒の履歴確認 |
| レッスン別 | Table | レッスン番号でグループ化 | 教材改善時に参照 |
| 改善が必要 | Table | 教材改善メモが空でない | 教材を直すべき箇所を抽出 |

### サンプルデータ

| タイトル | 日付 | 生徒 | レッスン番号 | 生徒の反応 | T&E度 | つまづいた箇所 | 振り返り | 気づいたこと |
|---------|------|------|------------|-----------|-------|-------------|---------|------------|
| 2026-03-09_田中太郎_L1 | 3/9 | 田中太郎 | Lesson 1 | ヒントで解決 | ★★ | Part挿入のメニューが見つからなかった | 「最初は難しかったけどCtrl+Dを見つけて嬉しかった」 | Homeタブの位置を最初に確認させた方がいい |

## DB2: 生徒カルテDB

生徒ごとの成長を一目で把握する。

### カラム定義

| # | カラム名 | 型 | 選択肢/設定 | 説明 |
|---|---------|------|-----------|------|
| 1 | 生徒名 | Title | - | |
| 2 | 学年 | Select | 小5 / 小6 / 中1 / 中2 | |
| 3 | 開始日 | Date | - | スクール開始日 |
| 4 | 現在のレッスン | Select | Lesson 1〜Lesson 18 | 今どこまで進んでいるか |
| 5 | 授業ログ | Relation | → 授業ログDB | 過去の全レッスンログ |
| 6 | 累計レッスン数 | Rollup | 授業ログの件数をカウント | 何回授業したか |
| 7 | 自力解決率 | Rollup/Formula | 授業ログの「自力で解決」の割合 | トライ&エラー力の指標 |
| 8 | 得意なこと | Text | - | |
| 9 | 苦手・注意点 | Text | - | |
| 10 | 性格メモ | Text | - | 「負けず嫌い」「慎重派」など |
| 11 | 保護者メモ | Text | - | 保護者から聞いた情報 |
| 12 | ステータス | Select | 受講中 / 休会中 / 卒業 | |

### ビュー設計

| ビュー名 | 種類 | 用途 |
|---------|------|------|
| 受講中の生徒 | Table (フィルタ: 受講中) | メインビュー |
| カード形式 | Gallery | 生徒一覧を見やすく |

## 移行手順

### Phase 1: 新DB作成
1. Notion で「授業ログDB」を新規作成（上記カラムで）
2. Notion で「生徒カルテDB」を新規作成
3. Relation でお互いを接続

### Phase 2: 既存データ移行
1. レッスンフィードバックDB の1件（Lesson 1）を授業ログDBに手動コピー
2. つまづきポイントDB の7件を授業ログDBに統合
3. 既存の生徒管理の情報を生徒カルテDBに移行

### Phase 3: 旧DBアーカイブ
1. トラブルシューティングガイドDB → 削除（空のため）
2. 教え方の判断ロジックDB → 削除（空のため）
3. レッスンプラン改善ログDB → 削除（空のため）
4. レッスンフィードバックDB → アーカイブ（データあり）
5. つまづきポイントDB → アーカイブ（データあり）

### Phase 4: ホームページ更新
アイボックスホームのリンクを新しい構造に更新:
```
アイボックスホーム
├── 概要
├── カリキュラム
├── 授業ログDB  ← 新
├── 生徒カルテDB  ← 新
└── （アーカイブ）旧DB
```
